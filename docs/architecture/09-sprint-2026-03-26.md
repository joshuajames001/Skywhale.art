## Issues dokončené

### GF-141: Gumroad → Stripe migrace

**Problém:** Gumroad jako platební brána — omezené API, chybějící webhook kontrola, vendor lock.

**Řešení:**
- `stripe-webhook/index.ts` — Stripe signature verification (`constructEventAsync`), handler pro `checkout.session.completed` (one-time + subscription) a `invoice.payment_succeeded` (renewal). Deduplikace přes `transactions.stripe_session_id`. Service role key pro admin operace.
- `create-checkout-session/index.ts` — auth required, vytváří Stripe Checkout session. Origin čte z request headeru (localhost:5173 lokálně, skywhale.art v produkci). Metadata: `user_id` + `energy_amount` pro webhook.
- `src/features/store/constants.ts` — centralizované Stripe price IDs (3 one-time, 4×2 subscription)
- `useEnergy.ts` — `GUMROAD_URLS` + `buyPackage()` → `purchaseEnergy()` přes `invokeEdgeFunction`
- `EnergyStore.tsx` — priceId z konstant, `purchaseEnergy(priceId, mode)`
- `SubscriptionCard.tsx` — zjednodušený `onBuy: () => void`

**Smazáno:** `supabase/functions/gumroad-webhook/` (celá složka)

### GF-143: Feedback Board → Email Form

**Problém:** Feedback nástěnka s community čtením/psaním je zbytečně komplexní. Nikdo ji nepoužívá k čtení. Stačí jednoduchý formulář.

**Řešení:**
- `send-feedback/index.ts` — auth + Resend API. From: `RESEND_FROM_EMAIL`, To: `FEEDBACK_EMAIL`. Plaintext s emailem uživatele + timestamp.
- `FeedbackForm.tsx` — modal s textarea (max 1000 znaků), success stav s velrybou, error s retry
- `useFeedbackForm.ts` — hook s `invokeEdgeFunction` (Three-Layer Rule)
- Route `/feedback` — lazy loaded (dříve direct import)

**Smazáno:** `FeedbackBoard.tsx`, `useFeedbackData.ts`

## Architektonické poznámky

### Stripe webhook pattern
- Raw body (`req.text()`) nutný pro signature verification — nelze parsovat jako JSON předem
- `constructEventAsync` (async verze) — vyžadováno v Deno runtime
- Subscription metadata: nastavována přes `subscription_data.metadata` v checkout session, čtena z `subscription.metadata` při renewal
- First invoice (`billing_reason: 'subscription_create'`) se skipuje — už handled checkout.session.completed

### Env vars (Supabase Secrets)
| Secret | Funkce |
|---|---|
| `STRIPE_SECRET_KEY` | stripe-webhook, create-checkout-session |
| `STRIPE_WEBHOOK_SECRET` | stripe-webhook |
| `RESEND_API_KEY` | send-feedback |
| `RESEND_FROM_EMAIL` | send-feedback (from address) |
| `FEEDBACK_EMAIL` | send-feedback (recipient) |
